# OKR Framework Reference
# OKR方法论参考指南

## What is OKR?

**OKR** = Objectives and Key Results (目标与关键结果)

- **Objective (目标)**: 定性的、鼓舞人心的、有时间限制的目标
- **Key Result (关键结果)**: 量化的、可衡量的、有明确成功标准的结果

## OKR vs KPI

| 维度 | OKR | KPI |
|------|-----|-----|
| **目的** | 驱动变革和增长 | 维持和衡量现状 |
| **周期** | 季度（灵活调整） | 年度（相对固定） |
| **目标设定** | 自下而上 + 自上而下 | 自上而下 |
| **完成度预期** | 60-70%（挑战性） | 100%（达标） |
| **公开性** | 全公司透明 | 通常保密 |

## OKR Writing Formula

### Objective Formula
```
[动词] + [目标描述] + [时间/地点限定]

例子:
✅ "在Q2成为中国市场份额第一的SaaS产品"
❌ "提升市场份额"（太模糊）
```

### Key Result Formula
```
[起始状态] → [目标状态] ([具体指标])

例子:
✅ "月活用户从50万增长到80万"
✅ "NPS从45分提升到65分"
✅ "产品Bug率从5%降低到2%以下"

❌ "提升用户体验"（无法衡量）
❌ "改进产品质量"（无具体指标）
```

## SMART Checklist for Key Results

- [ ] **Specific** - 具体明确，没有歧义
- [ ] **Measurable** - 可量化，有明确数值
- [ ] **Achievable** - 有挑战但可实现（60-70%信心度）
- [ ] **Relevant** - 与Objective强相关
- [ ] **Time-bound** - 有明确时间期限

## OKR Levels

### Company Level (公司级)
```
Objective: 成为亚洲领先的企业级AI服务平台
Key Results:
  - ARR从1亿增长到3亿
  - 付费企业客户数从500家增至1500家
  - NPS达到70分以上
```

### Department Level (部门级)
```
产品部 Objective: 打造业界最佳的AI对话产品体验

Key Results:
  - DAU从10万增至30万
  - 用户留存率（7日）从40%提升至60%
  - 产品推荐NPS达到80分
```

### Team Level (团队级)
```
研发团队 Objective: 大幅提升产品稳定性和性能

Key Results:
  - 系统可用性从99.5%提升至99.9%
  - 平均响应时间从500ms降至200ms
  - P0/P1 Bug率低于1%
```

### Individual Level (个人级)
```
产品经理 Objective: 成功推出并验证新功能的市场价值

Key Results:
  - 完成3个新功能的MVP并上线
  - 新功能用户采用率达到40%
  - 收集并分析至少100份用户反馈
```

## OKR Cascading (目标对齐)

### Vertical Alignment (垂直对齐)
```
公司 OKR
    ↓
部门 OKR (如何支撑公司目标?)
    ↓
团队 OKR (如何支撑部门目标?)
    ↓
个人 OKR (如何支撑团队目标?)
```

### Horizontal Alignment (水平对齐)
```
产品部 OKR ←→ 研发部 OKR ←→ 市场部 OKR
(跨部门依赖和协作)
```

## OKR Setting Process

### Step 1: Top-Down (自上而下)
1. 公司领导层设定公司OKR
2. 各部门基于公司OKR草拟部门OKR
3. 团队和个人基于部门OKR草拟自己的OKR

### Step 2: Bottom-Up (自下而上)
1. 个人和团队提出自己的创新OKR
2. 与上级讨论对齐
3. 整合到部门和公司OKR

### Step 3: Alignment Check (对齐检查)
- 每个下级OKR必须能回答"这如何支撑上级OKR?"
- 识别跨部门依赖
- 解决冲突和重叠

### Step 4: Finalization (最终确定)
- 全员公开OKR
- 确认资源分配
- 建立跟踪机制

## OKR Scoring (评分)

### Scoring Scale
- **0.0-0.3**: 未达成，需要反思
- **0.4-0.6**: 有进展但未完全达成
- **0.7-0.8**: 达成预期（理想区间）
- **0.9-1.0**: 超额完成（目标可能设定过低）

### Scoring Example
```
Objective: 显著提升产品用户留存率

KR1: 7日留存率从40%提升至60%
  实际: 52%
  得分: 0.6  (52-40)/(60-40) = 0.6

KR2: 月活用户从10万增至15万
  实际: 14万
  得分: 0.8  (14-10)/(15-10) = 0.8

KR3: NPS从50分提升至70分
  实际: 58分
  得分: 0.4  (58-50)/(70-50) = 0.4

Objective Score: (0.6 + 0.8 + 0.4) / 3 = 0.6
```

## OKR Review Cadence

### Weekly Check-in (周度检查)
- 5-10分钟快速回顾
- 当前进度 + 下周计划
- 识别阻碍因素

### Monthly Review (月度回顾)
- 30-60分钟深度讨论
- 详细进展分析
- 必要时调整策略

### Quarterly Review (季度复盘)
- 全面评估和打分
- 经验教训总结
- 制定下季度OKR

## Common Pitfalls (常见陷阱)

### ❌ Pitfall 1: Setting Too Many OKRs
**问题**: 目标过多，分散精力
**建议**: 3-5个 Objectives, 每个3-5个KRs

### ❌ Pitfall 2: KRs as Tasks
**错误**: KR="完成用户访谈"
**正确**: KR="基于30次用户访谈，识别3个核心痛点"

### ❌ Pitfall 3: Not Challenging Enough
**问题**: OKR设定过于保守
**建议**: 如果总能100%完成，说明目标不够挑战

### ❌ Pitfall 4: Lack of Alignment
**问题**: 各部门OKR各自为政
**建议**: 强制要求对齐检查和依赖识别

### ❌ Pitfall 5: Set and Forget
**问题**: 设定后就不管了
**建议**: 建立定期检查节奏

## OKR Templates

### Product Launch OKR
```
O: 成功推出X产品并获得市场验证

KR1: 产品上线并获得1000个付费用户
KR2: 用户满意度达到4.5/5星
KR3: 月度流失率低于5%
KR4: 实现50万元月度经常性收入(MRR)
```

### Team Growth OKR
```
O: 建立高效协作的产品团队

KR1: 招聘并onboard 5名产品经理
KR2: 团队满意度调查达到85分
KR3: 完成3次团队培训，参与率>90%
KR4: 建立标准化产品流程，使用率100%
```

### Efficiency Improvement OKR
```
O: 大幅提升研发效率

KR1: 需求交付周期从4周缩短至2周
KR2: 代码Review时长从48小时缩短至24小时
KR3: 自动化测试覆盖率从60%提升至85%
KR4: 生产环境Bug数量减少50%
```
