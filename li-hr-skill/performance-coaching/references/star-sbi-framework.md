# STAR & SBI Feedback Frameworks
# STAR和SBI反馈方法论

## STAR Method for Behavioral Feedback

**STAR** = Situation, Task, Action, Result

用于描述具体行为和结果的结构化方法。

### Framework Structure

```
Situation (情境)
  ↓
Task (任务)
  ↓
Action (行动)
  ↓
Result (结果)
```

### When to Use STAR
- 绩效评估
- 行为面试
- 成功案例分享
- 项目复盘
- 晋升答辩

### STAR Example - Positive Feedback

```
Situation: 上周五下午5点，客户突然报告系统关键bug，影响订单处理

Task: 需要在周末前修复bug，避免影响周一的业务高峰

Action:
- 你立即组织了跨部门紧急会议
- 用2小时定位到根本原因
- 协调研发、测试、运维团队加班修复
- 晚上10点完成修复并部署上线
- 主动向客户说明情况并致歉

Result:
- Bug在8小时内完全解决
- 避免了估计50万元的业务损失
- 客户对快速响应表示高度认可
- 团队凝聚力显著提升
```

### STAR Example - Constructive Feedback

```
Situation: 上月的产品发布会，你作为主讲人进行演示

Task: 向100+客户展示新功能，争取现场签约

Action:
- 演示PPT准备充分，功能讲解清晰
- 但演示时出现3次技术故障，现场尴尬
- 对客户问题的回答不够有信心
- 时间控制不佳，超时15分钟

Result:
- 客户反馈产品不错，但对稳定性有顾虑
- 现场签约率仅20%，低于预期50%
- 部分客户要求进一步技术验证

Improvement Suggestions:
- 提前进行多次完整演练，测试设备
- 准备备用方案应对技术故障
- 加强Q&A环节的准备，建立标准答案库
- 严格控制每个环节的时间
```

## SBI Method for Direct Feedback

**SBI** = Situation, Behavior, Impact

更简洁直接的反馈方法，聚焦于可观察的行为和影响。

### Framework Structure

```
Situation (情境)
  ↓
Behavior (行为)
  ↓
Impact (影响)
```

### When to Use SBI
- 日常快速反馈
- 1-on-1 conversation
- 实时辅导
- 纠正不当行为
- 及时表扬

### SBI Example - Positive Feedback

**Example 1:**
```
Situation: 在昨天的客户会议上

Behavior: 我注意到当客户提出质疑时，你没有立即辩解，
         而是先认真倾听，记录要点，然后有条理地回应

Impact: 这让客户感受到被尊重，会议氛围从紧张转为合作，
       最终客户签署了合同
```

**Example 2:**
```
Situation: 在这周的项目协作中

Behavior: 你主动每天同步项目进展，提前识别风险，
         并协调资源解决问题

Impact: 项目按时交付，团队成员都觉得很有安全感，
       跨部门协作效率明显提升
```

### SBI Example - Constructive Feedback

**Example 1:**
```
Situation: 在今天的团队会议上

Behavior: 我观察到你在同事发言时，多次打断，并且看手机

Impact: 这让发言的同事感到不被尊重，可能影响他们未来分享想法的意愿，
       也会影响团队的心理安全感

What I'd like to see: 在会议中保持专注，让同事完整表达观点后再发表意见
```

**Example 2:**
```
Situation: 关于上周的项目延期

Behavior: 当我询问进度时，你说"快好了"，但没有给出具体时间，
         直到deadline前一天才告诉我无法按时完成

Impact: 这导致我无法提前调整资源，最终影响了客户交付，
       也让我对项目掌控感到担忧

What I'd like to see: 如果遇到困难，请及时告知具体情况和预计时间，
                     这样我们可以一起想办法解决
```

## STAR vs SBI Comparison

| 维度 | STAR | SBI |
|------|------|-----|
| **复杂度** | 较复杂，4个要素 | 简单，3个要素 |
| **时长** | 3-5分钟 | 1-2分钟 |
| **适用场景** | 正式评估、深度复盘 | 日常反馈、即时辅导 |
| **关注点** | 任务完成全过程 | 具体行为和影响 |
| **准备时间** | 需要准备 | 可以即兴 |
| **结果导向** | 强（包含Result） | 中（关注Impact） |

**选择建议:**
- 绩效评估、晋升答辩 → 用STAR
- 日常反馈、1-on-1 → 用SBI
- 项目复盘 → 用STAR
- 即时表扬/纠正 → 用SBI

## Feedback Best Practices

### 1. Timeliness (及时性)
✅ 在行为发生后24-48小时内给予反馈
❌ 等到年度评估时才说

### 2. Specificity (具体性)
✅ "在昨天的客户演示中，你三次引用了竞品数据来突出我们的优势"
❌ "你的演示很好"

### 3. Focus on Behavior (聚焦行为)
✅ "你在会议上打断了同事3次"
❌ "你很粗鲁"

### 4. Balance (平衡)
- 正面反馈 : 改进反馈 = 4:1 (理想比例)
- 不要总是批评，也不要只会表扬

### 5. Actionable (可行动)
✅ "下次客户会议前，请提前2天发送会议议程"
❌ "你需要更好地准备会议"

### 6. Two-Way Dialogue (双向对话)
- 给予反馈后，询问对方的看法
- 倾听对方的解释和背景
- 共同探讨改进方案

## Difficult Conversations Framework

当需要给予tough feedback时：

### 1. Prepare (准备)
- 收集具体事实和数据
- 明确期望的改进结果
- 预想对方可能的反应
- 选择合适的时间和地点

### 2. Open with Care (开场)
```
"我想和你聊聊关于[具体事项]，这对你的发展很重要。
我希望这是一次坦诚、建设性的对话。"
```

### 3. State Facts (陈述事实)
- 使用STAR或SBI框架
- 避免评判性语言
- 关注可观察的行为

### 4. Listen Actively (积极倾听)
- 给对方充分的表达空间
- 理解对方的视角和困难
- 展现同理心

### 5. Collaborate on Solutions (共同解决)
- 问"你觉得可以怎么改进?"
- 提供具体的支持和资源
- 设定明确的跟进计划

### 6. Follow Up (跟进)
- 约定检查进度的时间
- 持续关注和支持
- 认可改进的努力

## Feedback Templates

### Template 1: Positive Reinforcement
```
[Name], I want to recognize your [specific behavior] in [situation].

This had a [positive impact] because [reason].

Keep up the great work, and thank you for [contribution].
```

### Template 2: Constructive Feedback
```
[Name], I'd like to discuss [specific situation].

I observed that [specific behavior].

This had [impact/consequence].

Going forward, I'd like to see [desired behavior].

How can I support you in making this change?
```

### Template 3: Performance Improvement
```
[Name], we need to address [performance issue].

Here are the specific concerns:
- [Specific example 1]
- [Specific example 2]

The expected standard is [clear expectation].

Let's create a plan together:
- Action 1: [具体行动]
- Action 2: [具体行动]
- Check-in: [时间]

I'm here to support you. What do you need from me?
```

## Common Mistakes to Avoid

### ❌ Mistake 1: Sandwich Feedback
**问题**: Positive → Negative → Positive (三明治反馈法)
**为什么不好**: 稀释了核心信息，对方记不住重点
**替代**: 如果是constructive feedback，就直接说

### ❌ Mistake 2: Vague Feedback
**错误**: "你需要更积极主动"
**改进**: "在项目会议中，请主动分享你的进展和需要的支持"

### ❌ Mistake 3: Feedback in Public
**错误**: 在团队会议上批评个人
**改进**: 表扬可以公开，批评应私下

### ❌ Mistake 4: Delayed Feedback
**错误**: 积累问题到年度评估时一起说
**改进**: 及时反馈，小问题不要拖成大问题

### ❌ Mistake 5: Emotional Feedback
**错误**: 在愤怒或沮丧时给反馈
**改进**: 冷静下来后，基于事实给反馈

## Self-Reflection Questions

反馈前问自己：

1. 我的反馈是基于事实还是假设?
2. 我是否准备了具体的例子?
3. 我的目的是帮助对方成长还是发泄情绪?
4. 时间和地点合适吗?
5. 我准备好倾听对方的观点了吗?
6. 我能提供哪些具体的支持?
